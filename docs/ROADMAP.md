# Eat it roadmap

## Stage 1: Persistent core data

- Node.js API and SQLite database.
- Fridge and shopping list synchronization.
- Create, edit, consume, move, complete, and delete operations.
- Loading and network error states.
- Web and API deployment instructions.

## Stage 2: Accounts and households

- Email/password registration and sign-in.
- Password recovery and account deletion.
- Google sign-in.
- Household creation, partner invitations, roles, and shared data.
- Move the current anonymous data into the first household.

## Stage 3: Recipes and recommendations

- Recipe details, ingredients, and steps.
- User-created recipes, search, and filters.
- Ingredient matching against the fridge.
- Add missing ingredients to the shopping list.
- Prioritize products approaching expiration.

## Stage 4: Dish matching and notifications

- Dish cards, swipe history, and partner matches.
- Expiration notifications for web and Android.
- Notification preferences and quiet hours.

## Stage 5: Production hardening

- HTTPS, rate limiting, audit logging, and database backups.
- Monitoring, analytics, privacy policy, and terms.
- Import/export, accessibility review, offline support, and e2e tests.
- Signed Android release, versioning, AAB, and Google Play preparation.
